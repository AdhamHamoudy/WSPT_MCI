# WSPT-MCI Flow Shop Scheduler

## Overview
This project implements the **WSPT-MCI** (Weighted Shortest Processing Time with Minimum Cost Insertion) algorithm in C++. It is designed to find the optimal scheduling sequence for a set of jobs in a **Proportionate Permutation Flow Shop** environment, with the goal of minimizing the total weighted completion time (∑ w_j C_j).

Instead of heavily simulating machine queues and idle times, this program leverages the exact mathematical formula proven by **Shakhlevich et al. (1998)** to instantly calculate job completion times.

## Mathematical Foundation
Because the flow shop is *proportionate* (a job's processing time is identical across all machines), the completion time of any job `j` on the final machine `m` can be calculated using:

C_j(σ) = ∑ p_k + (m-1) * max{p_1, ..., p_j}

This formula turns what would be a complex simulation into an efficient O(n^2) insertion algorithm.

## File Structure
* `WSPT-MCI.cpp`: The main C++ source code containing the sorting logic and the insertion algorithm.
* `jobs.txt`: The dynamic input file containing the machine count and job data.
* `Makefile`: A script to easily compile and execute the program via the terminal.

## Input Format (`jobs.txt`)
The program reads data dynamically from a text file named `jobs.txt` located in the same directory. 

* The **first line** must contain a single integer: the number of machines (m).
* Every **subsequent line** represents a single job formatted as three integers separated by spaces: `[Job_ID] [Processing_Time] [Weight]`.

**Example `jobs.txt`:**
4
1 2 4
2 5 6
3 7 9

## How to Compile and Run
This project includes a `Makefile` for easy execution on macOS, Linux, or WSL.

1. Open your terminal and navigate to the project folder.
2. Ensure your `jobs.txt` file is populated with your test data.
3. Run the following command to compile and execute the program instantly:
   make run

**Other available commands:**
* `make` - Compiles the `WSPT-MCI.cpp` file into an executable.
* `make clean` - Deletes the compiled executable to keep the directory clean.

## Output
The program will display:
1. The total number of jobs and machines loaded.
2. The initial sequence sorted purely by the WSPT rule (descending w/p ratio).
3. The final mathematically optimal sequence generated by the MCI logic.
4. The minimized total weighted completion cost.
